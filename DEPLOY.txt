================================================================
  PIJU COIN WEBSITE — DEPLOYMENT GUIDE
================================================================

This project can be deployed in two ways:
  A) Cloudflare Pages (recommended — free, fast, global CDN)
  B) Replit (already running if you're reading this there)

================================================================
  A) DEPLOYING TO CLOUDFLARE PAGES
================================================================

REQUIREMENTS:
  - A free Cloudflare account (cloudflare.com)
  - Your GitHub repo connected to Cloudflare
  - An Anthropic API key (get one at console.anthropic.com)

STEP 1 — Connect your GitHub repo to Cloudflare Pages
  1. Log in to dash.cloudflare.com
  2. Go to "Workers & Pages" in the left sidebar
  3. Click "Create" then "Pages" tab
  4. Click "Connect to Git"
  5. Authorize Cloudflare to access your GitHub account
  6. Select the repository: Harmain11/piju
  7. Click "Begin setup"

STEP 2 — Configure the build settings
  On the build settings screen:
    Framework preset:    None
    Build command:       (leave empty)
    Build output dir:    /  (a single forward slash — the root)
  Click "Save and Deploy"

STEP 3 — Add your Anthropic API key (IMPORTANT)
  This is required for the chatbot to work.

  1. After the first deploy finishes, go to your Pages project
  2. Click "Settings" in the top tabs
  3. Click "Environment variables" in the left sidebar
  4. Click "Add variable"
  5. Set:
       Variable name:  ANTHROPIC_API_KEY
       Value:          your-api-key-here   (paste your actual key)
  6. Make sure to set it for both "Production" and "Preview"
  7. Click "Save"
  8. Go back to "Deployments" and click "Retry deployment" (or push
     any small change to trigger a new deploy)

  Where to get an Anthropic API key:
    - Go to console.anthropic.com
    - Sign in or create an account
    - Navigate to "API Keys" in the left menu
    - Click "Create Key", give it a name, copy the key
    - Paste it into the Cloudflare environment variable above

STEP 4 — Your site is live!
  Cloudflare will give you a URL like:
    https://piju-xxxx.pages.dev

  You can also add a custom domain in Pages > Custom Domains.

HOW IT WORKS ON CLOUDFLARE:
  - The HTML, CSS, JS, and assets are served by Cloudflare Pages
    (fast global CDN — no server needed for static files)
  - The chatbot API (/api/chat) runs as a Cloudflare Function
    (file: functions/api/chat.js) — no Python server needed
  - The ANTHROPIC_API_KEY environment variable is read by the
    Cloudflare Function at runtime, so it stays secret

================================================================
  B) DEPLOYING ON REPLIT
================================================================

STEP 1 — Open the project in Replit

STEP 2 — Add your API key
  1. Click the "Secrets" tab (padlock icon) in the left sidebar
  2. Add a new secret:
       Key:    ANTHROPIC_API_KEY
       Value:  your-api-key-here
  3. Click "Add Secret"

STEP 3 — Run the project
  The workflow "Start application" runs:
    python3 server.py
  This starts a Flask server on port 5000 that serves the site
  and handles the chatbot API.

STEP 4 — Deploy to production
  Click the "Deploy" button in the top-right corner of Replit.
  Choose "Autoscale" deployment. The API key from Secrets is
  automatically available in production.

================================================================
  TROUBLESHOOTING
================================================================

Chatbot says "Network error" or doesn't respond:
  - Check that ANTHROPIC_API_KEY is set correctly
  - Make sure the key has not expired or been revoked
  - On Cloudflare: redeploy after adding the environment variable

Chatbot works locally but not after deploy:
  - On Cloudflare: the env variable must be set in the Pages
    dashboard, NOT in a .env file (that won't be read)
  - Trigger a fresh deployment after adding the variable
